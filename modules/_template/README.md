# Module Template

Use this folder as a starting structure for new architecture modules.

Copy the files into a new module folder and fill in only confirmed business rules, assumptions, and open questions.

## API Documentation

Use `api-contracts.md` for domain-specific interface contract concepts.

Use `openapi-contracts.md` for implementation-oriented API specification details, including endpoint inventory, request and response schemas, error models, authentication, throttling, pagination, versioning, and OpenAPI snippets.

## Event Documentation

Use `events.md` for the module event catalog and event modeling notes.

Use `event-contracts.md` for event interface contracts between modules, including payload structure, producers, consumers, retry behavior, versioning, and audit considerations.
